# Playbook: Backend Issue Execution

1. Read `AGENTS.md`, `docs/README.md`, and relevant plugin knowledge files.
2. Restate issue goal, scope, non-goals, and risks.
3. Identify affected modules.
4. Identify migration/API/security/concurrency impact.
5. Implement minimal scoped change.
6. Add required tests.
7. Update docs/OpenAPI/migrations when needed.
8. Run checks.
9. Summarize using the required final response shape.

Do not continue into the next issue.
