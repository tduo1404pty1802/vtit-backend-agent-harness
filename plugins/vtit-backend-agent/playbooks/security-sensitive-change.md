# Playbook: Security-Sensitive Change

Use this for auth, identity, evidence, audit, permissions, and case visibility.

1. Identify protected data.
2. Identify actor roles.
3. Identify ABAC attributes.
4. Verify response fields.
5. Add deny-by-default tests.
6. Add audit where required.
7. Check logs/errors do not leak sensitive data.

Stop if the permission matrix does not cover the action.
